Estrutura Básica de uma Página Web

Index / Exemplos de pastas

As pastas assets, styles e scripts são fundamentais para o desenvolvimento de websites e aplicações web modernas a gente já aprendeu. Cada uma delas possui uma função específica, permitindo que desenvolvedores organizem e gerenciem os arquivos do projeto de forma eficiente.

Para entender melhor a diferença entre essas pastas, vamos dar uma olhada em alguns exemplos práticos:

assets

  • background.jpg
  • cover.png
  • musica.mp3
  • efeito_sonoro.ogg
  • video.mp4

styles

  • header.css
  • index.css
  • global.css
  • perfil.css
  • efeitos.css
  • animacoes.css

scripts

  • script.js
  • events.js
  • dialog.js
  • login_user.js
  • interactions.js

Apesar das estruturas de pastas assets, styles e scripts estarem fixadas nos exemplos acima, elas podem variar dependendo do contexto de desenvolvimento. Por exemplo, se o desenvolvimento for voltado para uma loja virtual, as pastas assets podem conter imagens dos produtos, logotipos da empresa, ícones de pagamento, etc. Já as pastas styles podem conter arquivos CSS para a estilização da página inicial, de categorias de produtos, de páginas de checkout, etc. E as pastas scripts podem conter arquivos JavaScript para a interatividade com o usuário, validação de formulários, etc.

Em um contexto de desenvolvimento de um jogo, por sua vez, as pastas assets podem conter imagens dos personagens, animações, músicas de fundo, efeitos sonoros, etc. As pastas styles podem conter arquivos CSS para a estilização dos menus, barras de progresso, janelas de diálogo, etc. E as pastas scripts podem conter arquivos JavaScript para a lógica do jogo, interatividade com o usuário, tratamento de colisões, etc.
